> 안녕하세요! 🎉 프론트엔드 개발자를 꿈꾸는 **최원석**입니다.
> `Next.js`를 활용해 블로그를 직접 만들어보았습니다. 기록의 중요성을 느끼며, 개발자로서 꾸준히 학습 내용을 정리하고 공유하고자 블로그를 시작하게 되었습니다. 제가 만든 블로그인 만큼 더 애정을 가지고 꾸준히 글을 쓸 수 있을 것 같다는 기대도 있습니다. 앞으로는 꾸준히 글을 올리며 학습 기록을 이어갈 계획입니다. 스스로와의 약속인 만큼 성실히 지켜보려 합니다.😊

### 📌 구현 기능

---

- 게시글 목록 페이지 (전체, 카테고리별) ✅
- 게시글 상세 페이지 ✅
- 다크/라이트 모드 ✅
- 댓글 기능 ✅
- TOC(Table of Contents)
- 상단 progress-bar

위의 기능은 필수로 넣을 예정이고 사용해보면서 추가적인 기능 또한 꾸준히 개발해보려고 합니다.

### 📌 기술 스택

---

#### ✅ Next.js

최근에 Next.js를 점점 더 많이 사용해가고 있는 추세이고, 회사 지원 공고에서도 보면 요즘은 Next.js를 안 찾아보기가 힘든 것 같습니다. 이번 기회를 통해 좀 더 익숙해지고 아직 제대로 다뤄보지 못한 app router 방식을 적용해보고자 합니다.

#### ✅ Tailwind CSS + shadcn/ui

utility-first-css인 tailwind를 적용해보고자 합니다. 최근에 많이 쓰이고 개발속도가 빠른 장점이 있는것으로 알고 있고 아직 한번도 경험해보지 못해서 css tool은 tailwind로 정했습니다. 추가로 component collection인 shadcn/ui 또한 적용해보고자 합니다. tailwind 기반으로 만들어졌고 빠른 개발을 위해 적용하기로 결정했습니다.

#### ✅ mdx

마크다운 확장인 mdx를 활용하여 글을 작성할 것 입니다. mdx는 마크다운과 jsx를 같이 사용할 수 있습니다. 내부 디렉토리에 저장할것이고 next를 활용한 블로그에서 주로 선택되는 방법 입니다.

#### ✅ gray matter + Next-MDX-Remote

gray-matter는 MDX 파일의 상단에 있는 Front Matter(메타데이터)를 읽어와 콘텐츠와 분리하는 역할을 합니다.
Next-MDX-remote는 분리된 MDX 본문을 React 컴포넌트로 변환하고 렌더링합니다.

#### ✅ Giscus

Github Discuss 시스템을 기반으로 작동하는 댓글 서비스입니다.
깔끔한 UI를 제공하여 댓글 시스템은 Giscus를 도입하겠습니다.
